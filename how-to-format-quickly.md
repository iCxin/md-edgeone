# 如何最快给一篇推文排版？

## 🚀 介绍：微信Markdown编辑器魔改版

基于@doocs的原项目二次开发的**微信Markdown编辑器 | www.cxin.net魔改版**，专注于提供**极致简洁的Markdown编辑体验**。

> ⚡ **核心理念**：写作应该专注于内容，而不是在排版工具上浪费时间。

## 🎯 一键式排版系统

### 📊 基础样式（5秒搞定）

| 功能 | 快捷选项 | 预设效果 |
|------|----------|----------|
| **主题** | 3种预设 | 经典/暗色/高亮 |
| **字体** | 3种字体 | 微软雅黑/思源黑体/PingFang |
| **字号** | 5档调节 | 12px → 16px → 20px |
| **主题色** | 6种预设 + 自定义 | 蓝/绿/红/紫/橙/自定义 |

### 🔢 公式排版（3秒搞定）

| 类型 | 预设大小 | 适用场景 |
|------|----------|----------|
| **行内公式** | 0.9em → 1.3em | 与文字完美融合 |
| **块级公式** | 0.95em → 1.35em | 突出显示，保持协调 |
| **背景色** | 6种预设 | 灰色/彩色/透明背景 |

### 📝 文本排版（5秒搞定）

| 项目 | 预设范围 | 推荐设置 |
|------|----------|----------|
| **标题行高** | 1.0 → 1.4 | **1.2**（清晰不拥挤） |
| **正文行高** | 1.2 → 1.6 | **1.5**（舒适阅读） |
| **标题间距** | 0.5em → 3em | **段前2em，段后1em** |
| **段落间距** | 1em → 3em | **前后各1.5em**（呼吸感） |

### 🎨 高级样式（2秒搞定）

| 元素 | 控制选项 | 一键设置 |
|------|----------|----------|
| **引用块** | 边框宽度 + 背景色 | **4px + 浅灰背景** |
| **代码块** | 圆角大小 | **8px**（现代感） |
| **表格** | 边框宽度 + 颜色 | **1px + #dfdfdf** |

## ⚡ 30秒排版工作流

### 第一步：打开编辑器（0秒）
```bash
# 启动编辑器
pnpm start
```

### 第二步：选择基础样式（5秒）
1. **主题** → 选择"经典"（最通用）
2. **字体** → 选择"微软雅黑"（微信友好）
3. **字号** → 选择"16px"（标准大小）
4. **主题色** → 选择"蓝色"（专业感）

### 第三步：设置公式（5秒）
1. **行内公式** → 选择"1.0em"（与文字一致）
2. **块级公式** → 选择"1.05em"（略大突出）
3. **公式背景** → 选择"透明"（简洁大方）

### 第四步：调整排版（10秒）
1. **标题行高** → 选择"1.2"
2. **正文行高** → 选择"1.5"
3. **标题间距** → 段前"2em"，段后"1em"
4. **段落间距** → 前后各"1.5em"

### 第五步：美化元素（5秒）
1. **引用块** → 边框"4px"，背景"浅灰"
2. **代码块** → 圆角"8px"
3. **表格** → 边框"1px"，颜色"#dfdfdf"

### 第六步：自定义微调（5秒）
- 如需特殊需求，使用自定义输入框
- 支持任意CSS值，精确控制

## 🎯 **核心观点：不要浪费时间在刻意排版上**

### ❌ **常见的排版陷阱**

1. **过度调整**：花半小时调一个像素的间距
2. **频繁切换**：在5个主题间来回选择
3. **像素级追求**：为0.1em的大小纠结半天
4. **反复试错**：没有明确目标地不断尝试

### ✅ **高效的排版原则**

1. **一次设置**：找到适合自己的组合，保存为默认
2. **专注内容**：排版只是工具，内容才是核心
3. **适度即可**：80分的效果胜过100分的耗时
4. **批量应用**：用预设快速处理，不用每次都调

写作 → 套用预设 → 检查可读性 → 一键复制 → 发布
